# CLASS219 강사용 해설서

- **교과목명**: 자연어 및 음성 데이터 활용 및 모델 개발  
- **차시 주제**: 실전 데이터셋 설계  
- **일정**: Day 28 / 3교시 (1일 8시간 운영 기준)  
- **실습 포커스**: 데이터프레임/전처리

## 수업 진행 가이드 (10/30/10)

### 1) 설명 10분
- 오늘 목표/산출물: `classXXX_assignment.py`의 TODO를 완성하여 정상 실행
- 개념 포인트: 데이터프레임/전처리에서 **왜 이 패턴을 쓰는지**(실무 관점) 강조
- 데모(짧게): 최종 출력/저장 파일 경로를 먼저 보여주고 “이걸 만들자”로 동기 부여

### 2) 실습 30분
- 학생 작업: TODO 1~3 구현
- 체크포인트:
  - 10분: TODO1 동작 확인(출력 확인)
  - 20분: TODO2 예외 처리/엣지 케이스 확인
  - 30분: TODO3 파일 저장/산출물 생성 확인

### 3) 정리 10분
- 리뷰 질문
  - “오늘 구현에서 가장 헷갈렸던 지점은?”
  - “실무에서 이 코드가 깨지는 케이스는?”
- 리팩토링 1개만 제안(짧게): 함수 분리/타입 힌트/테스트 포인트 등

## 차시 목표(원문)
- 실전 데이터셋 설계 핵심 개념 이해
- 실전 데이터셋 설계 관련 코드/도구 적용
- 실전 데이터셋 설계 결과 점검 및 다음 차시 연결

## 예상 산출물/정답 포인트
- `classXXX_solution.py`가 참고 정답
- 출력이 일부 달라도 **핵심 조건 충족**(TODO 요구사항)하면 통과

## 자주 발생하는 실수
- 경로 처리: `Path(__file__).parent` 대신 현재 작업폴더에 저장해 파일 누락
- 예외처리: 광범위 `except:`로 문제 은폐
- 데이터/모델: 입력 차원(X shape) 혼동(특히 ML 카테고리)

## 채점(간단 루브릭)
- **필수(70%)**: TODO 요구사항 충족, 실행 오류 없음
- **가독성(20%)**: 함수/변수명 명확, 과한 중복 없음
- **확장(10%)**: 엣지케이스 처리, 로그/출력 정리, 주석/문서화


## 난이도별 과제

- **Basic(입문)**: `class219_assignment_basic.py`
  - 목표: 핵심 개념/기본 구현을 완료하고, 실행/출력까지 확인
  - 체크: TODO 구현 완료, 예외 없이 실행, 출력/파일 생성 확인

- **Advanced(심화)**: `class219_assignment_advanced.py`
  - 목표: 입력 검증 + 결과 구조화(JSON) + 간단 리포트 출력
  - 체크: 실패 케이스 처리(예외/가드), 결과 파일(`class219_advanced_result.json`) 생성, 코드 가독성

- **Challenge(챌린지)**: `class219_assignment_challenge.py`
  - 목표: 리팩토링 + 자체 테스트(assert) + 성능 측정/로그
  - 체크: 함수 분리, 테스트 3개 이상, 실행 시간 측정값 출력/기록

## 채점 루브릭(권장)

- 완성도(50): 요구사항 충족, 예외 없이 동작
- 코드 품질(30): 함수 분리, 네이밍, 주석/문서화
- 검증/테스트(20): assert/간단 테스트, 경계값 처리
