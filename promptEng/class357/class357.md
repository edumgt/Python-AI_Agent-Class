1. `기본 구조`는 role -> goal -> input -> output format -> constraints 순서로 정리합니다.
2. `입력 데이터 분리`는 템플릿 재사용성과 테스트 가능성을 높입니다.
3. `제약조건 명시`는 길이·금지규칙·톤 제어의 기준점입니다.

### 비유로 이해하기
